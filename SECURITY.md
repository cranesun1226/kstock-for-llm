# Security Policy

## 민감정보

OpenDART API 인증키와 로컬 `.env` 파일은 절대 커밋하지 않습니다. 인증키가 노출되었다고 의심되면 즉시 OpenDART에서 키를 재발급하거나 폐기한 뒤, Git 기록과 배포 위치를 점검해주세요.

## 취약점 신고

보안 취약점, secret 노출, 악용 가능한 파싱/파일 처리 문제를 발견했다면 공개 이슈에 민감정보나 재현 가능한 공격 payload를 올리지 말아주세요.

가능하면 GitHub의 private vulnerability reporting 기능을 사용해주세요. 해당 기능이 꺼져 있다면 공개 이슈에는 최소한의 설명만 남기고, 인증키·원문 데이터·개인정보·구체적인 악용 절차는 포함하지 마세요.

## 지원 범위

현재는 `main` 브랜치의 최신 코드만 보안 수정 대상으로 봅니다. 릴리스 태그가 생기면 지원 버전 정책을 이 문서에 갱신합니다.
